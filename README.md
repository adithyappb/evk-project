# EVK — Student Opportunity Pipeline

EVK is a multi-agent pipeline that turns noisy opportunity newsletters into short, **personalised** emails for individual students — with a mandatory **human-approval gate** in front of every send.

- **Ingestion** via [Inkbox](https://github.com/VectorlyApp/inkbox) (webhook or polling), with newsletter-footer sanitisation
- **Classification + extraction** via Vertex AI Gemini (or the `GOOGLE_API_KEY` Developer API) — retried x3, schema-validated, `temperature=0.1`
- **Fuzzy dedup** (type + deadline-window pre-filter → Jaccard similarity) so near-duplicate newsletters don't double-publish
- **Rule-based matching** of opportunities to students
- **Personalised drafts** generated by Gemini against a strict Pydantic schema, with student PII pseudonymised (hashed ID, age bands, region-only location) before any model call
- **Weekly digest agent** — top N picks per opted-in student, one draft per ISO week (idempotent)
- **Approval dashboard** (Jinja + Tailwind + HTMX) for a human to review / approve / reject
- **Distribution** back through Inkbox once approved — batched at 45 recipients, 0.2s spacing, daily per-process quota
- **Deadline reminders** on schedule (idempotent per `(student, opp, window)`)

> Ships with a **zero-credential local mode** — copy `.env.example` to `.env` and everything runs on a file-backed store, a pattern-matching stub Gemini, and a file-logged stub Inkbox. Flip `EVK_MODE=production` when you're ready for real services.

## 60-second quickstart (no credentials, no cloud)

### 1. Prerequisites

- Python **3.11+**
- [`uv`](https://docs.astral.sh/uv/) package manager (`pipx install uv` or `winget install astral-sh.uv`)

### 2. Install + configure

```bash
# clone or cd into the project
git clone <repo-url> google_evk_project
cd google_evk_project

# install pinned deps into a project venv
uv sync

# create your local env file FROM THE TEMPLATE (.env is gitignored)
cp   .env.example .env     # macOS / Linux
copy .env.example .env     # Windows PowerShell / cmd
```

> **`.env.example` is a PRE-PRODUCTION TEMPLATE ONLY.** It's checked into the repo and only ever contains stub placeholders (`ApiKey_local_stub`, `whsec_local_stub_signing_key_change_me`, …). Your real `.env` — and `secrets/`, `*.pem`, `*.key`, `service-account*.json` — are blocked by `.gitignore`. In production, inject real secrets through your platform's native mechanism (Cloud Run env vars, Secret Manager, k8s Secrets, GitHub Actions secrets, …) rather than a `.env` file in the container.

### 3. Seed + run

```bash
uv run evk seed               # loads 2 students + 10 opportunities into ./data
uv run evk serve              # http://localhost:8080
```

Open `http://localhost:8080`, click **Simulate newsletter** → a draft appears → click **Approve** → the send is logged to `data/sent.jsonl`.

### 4. Verify

```bash
uv run evk info                                  # show wiring
curl http://localhost:8080/healthz               # liveness  (cheap)
curl http://localhost:8080/health                # readiness (exercises repos + inkbox + gemini)
uv run pytest -q                                 # 160 tests
```

`evk info` should print:

```
        EVK wiring
┌────────────┬────────────┐
│ component  │ backend    │
├────────────┼────────────┤
│ mode       │ local      │
│ repos      │ local_json │
│ inkbox     │ stub       │
│ gemini     │ stub       │
└────────────┴────────────┘
```

> **Windows / OneDrive note.** If the project lives inside a OneDrive-synced folder (e.g. `OneDrive\Documents\...`), OneDrive will fight with `.venv` and `uv sync` will fail with `ERROR_CLOUD_FILE_INCOMPATIBLE_HARDLINKS` or `Access is denied` on `dist-info\licenses`. Run `. .\scripts\setup.ps1` once — it pins the venv to `%LOCALAPPDATA%\uv-venvs\` (outside OneDrive) and sets `UV_LINK_MODE=copy` for the session. `pyproject.toml` already sets `link-mode = "copy"` globally for the project.

## Configuration & secrets

| File              | Checked into git? | Purpose                                           |
| ----------------- | ----------------- | ------------------------------------------------- |
| `.env.example`    | **Yes**           | Pre-production template — stub values only        |
| `.env`            | **No** (gitignored) | Your local, developer-specific overrides        |
| `secrets/`        | **No** (gitignored) | Service-account JSON, signing keys, PEMs        |

`.gitignore` blocks every `.env*` file (except the template), `secrets/`, `*.pem`, `*.key`, and any `service-account*.json`. **Never commit a real credential; if one leaks, rotate it immediately.**

For production, the recommended layout is:

- `EVK_MODE=production`
- Set `GOOGLE_CLOUD_PROJECT`, `INKBOX_API_KEY`, `INKBOX_SIGNING_KEY`, `ADMIN_API_TOKEN`, and `PRIVACY_SALT` via your platform's secret store — not in a file.
- Mount the service-account JSON (if needed) via Secret Manager / volume, **not** via the image.
- Probe `/health` from your load balancer; 503 responses drain traffic automatically when Firestore / Inkbox / Gemini are unreachable.

## Architecture

```
 inbound newsletter ──▶ Inkbox mailbox ──webhook──▶ /webhooks/inkbox (HMAC)
                                                       │
                                                       ▼
   ┌──────────────────────────────────────────────────────────────┐
   │ IngestionAgent                                               │
   │   ├─ ClassifierAgent   (Gemini structured output)            │
   │   └─ PersonalizerAgent (score → Gemini copy → draft)         │
   └──────────────────────────────────────────────────────────────┘
                                                       │
                           writes to Firestore / local JSON
                                                       │
                                                       ▼
                        ┌───── Admin dashboard (Jinja+HTMX) ─────┐
                        │ human approves / rejects drafts         │
                        └─────────────────────────────────────────┘
                                                       │
                                                       ▼
               DistributorAgent  ──▶  Inkbox send (or stub log)
                                                       │
                                                       ▼
                     ReminderAgent (scheduled, idempotent)
```

**No outbound email is ever sent without a `DraftMessage.status == APPROVED`.**

## Modes

| `EVK_MODE` | Repos            | Inkbox           | Gemini                              |
| ---------- | ---------------- | ---------------- | ----------------------------------- |
| `local`    | File-backed JSON | File-logged stub | Regex + templated stub              |
| `local` + `GOOGLE_API_KEY`  | File-backed JSON | File-logged stub | **Real Gemini Developer API**      |
| `production` | Firestore      | Real Inkbox SDK  | Vertex AI Gemini (or Developer API) |

Switching modes is a single env var change — the dependency factory in `evk.factory` picks the right concrete client at request time.

## Admin dashboard

`GET /` serves a responsive dark-mode dashboard:

- Live counters (pending, sent today, opportunities, students) refresh every 5s
- One-click **Simulate newsletter**, **Poll Inkbox**, **Run reminders**, **Build digest**
- Tabbed drafts (pending / approved / sent / rejected / failed)
- Per-draft Approve (sends) / Reject (archives)

Behind the UI are REST endpoints — the dashboard is HTMX-on-top so each button round-trips a small fragment:

| Method | Path | Use |
| ------ | ---- | --- |
| `GET`  | `/`                        | Dashboard |
| `GET`  | `/healthz`                 | Cheap liveness probe (always 200) |
| `GET`  | `/health`                  | Deep readiness probe — checks repos + Inkbox + Gemini; 503 if any is down |
| `POST` | `/ui/simulate`             | Feed `seed/sample_newsletter.txt` into the pipeline |
| `POST` | `/ui/poll`                 | Pull unread from Inkbox |
| `POST` | `/ui/remind`               | Trigger reminder sweep |
| `POST` | `/ui/digest`               | Build this week's student digest drafts |
| `POST` | `/ui/drafts/{id}/approve`  | Approve + send |
| `POST` | `/ui/drafts/{id}/reject`   | Reject |
| `POST` | `/webhooks/inkbox`         | Inkbox inbound webhook (HMAC-signed) |
| `GET`  | `/admin/drafts`            | JSON list of drafts (filter by status) |
| `POST` | `/admin/drafts/{id}/approve` | Approve a draft (JSON) |
| `POST` | `/admin/drafts/{id}/reject`  | Reject a draft (JSON) |
| `GET`  | `/admin/opportunities`     | JSON list |
| `GET`  | `/admin/students`          | JSON list |
| `POST` | `/admin/students`          | Upsert a student (JSON) |
| `POST` | `/admin/poll`              | Trigger an Inkbox poll from JSON clients |

> All `/admin/*` routes share a single auth dependency. Set `ADMIN_API_TOKEN` in `.env` to require an `Authorization: Bearer <token>` header on every admin call (leave empty in dev to skip auth). In production, also front the service with Cloud Run IAM / IAP.

## Production switch-on

1. Provision a Google Cloud project with Vertex AI + Firestore enabled.
2. Service account with `Vertex AI User` + `Cloud Datastore User`, JSON key at `./secrets/service-account.json`.
3. Deploy the compound indexes: `gcloud firestore indexes composite create --config=firestore.indexes.json`.
4. Create an Inkbox agent, grab its API key + signing key.
5. In `.env`:
   ```
   EVK_MODE=production
   GOOGLE_CLOUD_PROJECT=your-gcp-project
   GOOGLE_CLOUD_LOCATION=us-central1
   INKBOX_API_KEY=ApiKey_real_xxx
   INKBOX_AGENT_HANDLE=your-agent
   INKBOX_SIGNING_KEY=whsec_real_xxx
   ADMIN_API_TOKEN=generate-a-long-random-string
   PRIVACY_SALT=another-long-random-string
   ```
6. `uv run evk seed && uv run evk serve`
7. Point Inkbox's webhook at `https://your-host/webhooks/inkbox`.
8. Set your orchestration layer (Cloud Run / Cloud Scheduler / k8s readiness) to probe `/health` — it returns 503 when Firestore, Inkbox, or Gemini are unreachable so traffic drains automatically.

If you'd rather poll than webhook: `uv run evk scheduler --poll-minutes 5 --remind-hours 6`.

## Safety rails

- **Idempotent everywhere**: inbound dedupe on Inkbox id, opportunity dedupe by content hash + fuzzy `(type, deadline±30d)` match, reminder dedupe by `(student, opp, window)`, digest dedupe by ISO week.
- **Approval-only send**: `DistributorAgent.send_one` refuses non-`APPROVED` drafts; new drafts default to `PENDING_APPROVAL`.
- **PII pseudonymisation**: every student fact handed to Gemini is routed through `evk.privacy` — salted-hash ID, age-band instead of birth year, region instead of street-level location.
- **Send pacing + circuit breaker**: batches capped at 45 recipients, 0.2s spacing between sends, daily per-process quota (`DELIVERY_DAILY_QUOTA`); exceeded quota aborts the run and leaves drafts in `APPROVED` for the next pass.
- **HMAC webhook verify** with ±5min skew window (`INKBOX_SIGNING_KEY`).
- **Retry with backoff** on Inkbox + Gemini calls (3 attempts, exponential).
- **Structured Gemini outputs** — Pydantic-schema-validated, `temperature=0.1`; malformed JSON raises `GeminiError`.
- **Admin surface is auth-gated** — set `ADMIN_API_TOKEN` to require `Authorization: Bearer …` on every `/admin/*` route (single router-level dep, no per-endpoint drift).
- **Batched writes**: Firestore repos commit in chunks of ≤500, matching the Firestore hard limit.
- **Deep readiness probe**: `/health` exercises Firestore, Inkbox, and Gemini and returns 503 if any fails — safe to wire to a load balancer.
- **Zero-credential local mode** so you can exercise the whole pipeline without a cloud account before flipping to production.

## FAQ

**Why Inkbox instead of the Gmail API?** Inkbox exposes a provider-neutral inbound/outbound API with first-class webhooks, HMAC signing, and agent-scoped mailboxes — so swapping to Gmail later only touches `inkbox_client.py`.

**Where is the real Inkbox SDK pinned?** Directly from GitHub: `inkbox @ git+https://github.com/VectorlyApp/inkbox@main#subdirectory=python` (see `pyproject.toml`).

**Why a dashboard *inside* the service?** Because the admin surface is tiny (approve / reject / simulate / counters) and co-locating it means no separate frontend deploy. A production-grade SPA can still be built against the same `/ui/*` fragments or the `/admin/*` JSON endpoints.

**How do I reset local state?** Delete the `data/` folder and rerun `evk seed`.
